# Тихий Тыл — Логика работы всех фич

> Документ для демонстрации. Описывает внутреннюю логику принятия решений, пороговые значения, формулы расчёта и пользовательские сценарии.

---

## 1. Шкала оценки состояния (эмодзи-шкала)

Медицинский стандарт: **1 = нет боли, 10 = невыносимая боль**.

| Эмодзи | Значение | Подпись | Что означает |
|--------|----------|---------|-------------|
| 😊 | 1 | Отлично | Всё хорошо, боли нет |
| 🙂 | 3 | Хорошо | Небольшой дискомфорт |
| 😐 | 5 | Так себе | Умеренная боль, порог для рекомендации техники |
| 🙁 | 7 | Плохо | Сильная боль |
| 😢 | 10 | Очень плохо | Невыносимая боль |

Клиент выбирает один из 5 вариантов (не произвольное число). Значение сохраняется как `pain_level` в БД.

## 2. Дневник настроения

### Флоу записи

```
Выбор режима (голос / текст)
    ↓
[Голос] Запись → Whisper STT → расшифровка
[Текст] Ввод текста
    ↓
Выбор эмодзи (оценка состояния 1–10)
    ↓
Сохранение записи в БД
    ↓
pain_level >= 5? ─── НЕТ → Перенаправление на «Мои записи»
    │
    ДА
    ↓
Запрос рекомендации техники (API /api/recommendations)
    ↓
Предложение техники (модалка) → Принял / Отказался
```

### Когда предлагается техника

Порог: `pain_level >= 5` (оценки 😐 5, 🙁 7, 😢 10).

При оценках 😊 1 и 🙂 3 — техника не предлагается, сразу переход к списку записей.

### Голосовая запись

- Формат: WebM (MediaRecorder API)
- Распознавание: Whisper base (локально на сервере)
- Результат: текстовая расшифровка сохраняется в поле `transcription`

## 3. Рекомендация техники (персонализация)

### Логика выбора (`recommendations.py`)

```
pain_level < 5 → техника не нужна (null)
    ↓
Есть ли в истории техника с helped=true?
    ↓
ДА → предложить её: «Тебе раньше помогало "Название". Попробуем?»
    ↓
НЕТ → стандартная дыхательная техника: «Начнём с дыхательной техники для быстрого успокоения»
```

**Как определяется «помогло»:**
- `helped = true` когда `pain_change <= -1` (боль уменьшилась хотя бы на 1 балл)
- При нескольких успешных техниках выбирается та, у которой больше `success_count` (количество раз, когда помогла)

**Фоллбэк:** если API рекомендаций недоступен, показывается статический блок с кнопками «Да/Нет» (без персонализации).

## 4. Техники самопомощи

### Категории

| Категория | Англ. ID | Количество техник |
|-----------|----------|-------------------|
| Дыхание | `breathing` | 2 |
| Заземление | `grounding` | 2 |
| Релаксация | `relaxation` | 2 |

### Последовательность после дневника

Первая техника — всегда **дыхательная** (если нет персонализации из истории). Если предлагается вторая — **заземление**.

Максимум 2 техники подряд (параметр `seq`).

### Плеер техники

- HTML5 Audio (play/pause, прогресс-бар, таймер)
- Текстовые инструкции
- Кнопка «Завершить досрочно» (пропуск аудио)
- По окончании аудио или досрочном завершении → оценка состояния после

### Оценка после техники

Клиент выбирает эмодзи (такая же шкала 1–10).

**Расчёт:**
```
pain_change = pain_after - pain_before
```

Отрицательный `pain_change` = улучшение (боль уменьшилась).

### Логика ветвления после техники

```
pain_change <= -2 (значительное улучшение)
    → «Отлично! Тебе стало значительно лучше» → Главная

pain_change <= -1 И это 1-я техника (seq < 2)
    → «Хорошо! Маленькие шаги. Хочешь ещё одну технику?»
    → Да → 2-я техника (заземление)
    → Нет → КБТ-флоу

pain_change > -1 И это 1-я техника (seq < 2)
    → «Эта техника не помогла. Попробуем другую?»
    → Да → 2-я техника (заземление)
    → Нет → КБТ-флоу

Любой результат после 2-й техники (seq = 2)
    → «Хорошо, есть улучшение» / «Техники не помогли»
    → КБТ-флоу
```

## 5. КБТ-сессии (когнитивно-поведенческая терапия)

### Когда запускается

КБТ-флоу запускается при любом **отказе от продолжения**:
- Отказ от техники в дневнике (кнопка «Нет, спасибо»)
- Отказ от 2-й техники после плеера
- Завершение 2-й техники (в любом случае)

### Полный флоу

```
«А хочешь мы просто поговорим?»
    ↓
НЕТ → «Помни, что я рядом. Ты можешь всегда вернуться.» → Главная
    ↓
ДА → 5 вопросов КБТ (по одному)
```

### 5 вопросов

| # | Вопрос | Тема |
|---|--------|------|
| 1 | Опиши свои эмоции не фильтруя: тревога, грусть, страх, злость, обида... | Эмоции |
| 2 | Какие мысли не дают тебе покоя, что ты всё время прокручиваешь в голове? | Мысли |
| 3 | Чего ты боишься? Страхи, сомнения, риски... Опиши их все. | Страхи |
| 4 | Чего ты сейчас хочешь? Опиши свои желания и потребности. | Желания |
| 5 | Что ты сейчас сделал? Расскажи о своих действиях которые ты предпринял. | Действия |

Каждый вопрос можно **пропустить** (кнопка «Пропустить вопрос»). Текстовое поле для ответа.

### После 5 вопросов — переоценка боли

Шкала 1–10 (числовые кнопки, не эмодзи). Вопрос: «На сколько плохо ты себя ощущаешь?»

### Поддерживающая фраза по результату

```
pain_change <= -2 (значительное улучшение):
    «Ты молодец, что поговорил о своих чувствах. Тебе уже лучше — это важно.»

pain_change <= -1 (небольшое улучшение):
    «Хорошо, что ты смог выразить свои мысли. Маленькие шаги — тоже шаги.»

pain_change >= 0 (без улучшения):
    «Спасибо, что поделился. Помни — ты не один, и обратиться к психологу не стыдно.»
```

### Что сохраняется в БД

- `pain_before`, `pain_after`, `pain_change`
- Привязка к записи дневника (если есть)
- Все 5 ответов (с пометкой «пропущено» если пропустили)

## 6. Профиль клиента

### Код-позывной

- Формат: **СЛОВО-ЦЦЦ** (пример: БУРЯ-472, ВОЛК-103)
- Словарь: 24 слова военно-природной тематики
- Срок действия: 7 дней
- Можно перегенерировать (старый деактивируется)
- Используется психологом для входа в дашборд

### Статистика клиента (только позитив)

| Показатель | Как считается |
|------------|--------------|
| Дней практик за неделю | COUNT(DISTINCT дат) записей за последние 7 дней |
| Тренд | Сравнение AVG(pain_level) за 3 дня vs 7–10 дней назад |
| Полезные техники | Техники где helped=true, ТОП-3 по количеству успехов |

**Логика тренда:**

```
avg_recent < avg_older - 1  →  «improving» (улучшение)
всё остальное               →  «stable» (стабильно)
```

**Важно:** тренд НИКОГДА не показывает ухудшение. Если средняя боль выросла — клиент видит «стабильно», не «хуже».

### Настройки

- Режим ввода по умолчанию: голос / текст

## 7. Дашборд психолога

### Вход

Психолог вводит код-позывной клиента. Проверяется:
- Код существует
- Код активен
- Не истёк срок (7 дней)

Каждый вход логируется (`access_logs`). Авторизация через `sessionStorage` (закрыл вкладку — вышел).

### Статистика (обзор)

| Карточка | Источник | Формула |
|----------|----------|---------|
| Всего записей | diary_entries | COUNT(*) |
| Первая запись | diary_entries | MIN(created_at) |
| Средняя боль | diary_entries | AVG(pain_level), округление до 1 знака |
| Последняя запись | diary_entries | MAX(created_at) |

**Отличие от клиента:** психолог видит ВСЮ статистику, включая негативную динамику.

### График динамики боли

- **Ось X:** даты (DD.MM.YYYY)
- **Ось Y:** уровень боли (1–10), стандартная ось (1 внизу, 10 вверху)
- **Данные:** `AVG(pain_level)` сгруппированные по `DATE(created_at)`
- **Визуал:** линейный график, цвет #4A90A4, заливка под линией
- **Интерпретация:** график идёт вниз = клиенту лучше

**Несколько записей в день:** усредняются. Если за 12 апреля две записи с болью 7 и 3, на графике будет точка 5.0.

### Таблица техник

| Колонка | Формула |
|---------|---------|
| Использований | COUNT(*) из technique_usage |
| Эффективность | (success_count / usage_count) * 100% |
| Изменение боли | AVG(pain_change), отрицательное = улучшение |

### КБТ-сессии

Психолог видит:
- Дату сессии
- Боль до → после (с цветовой индикацией: зелёный = улучшение, красный = ухудшение)
- Все 5 ответов (пропущенные показываются полупрозрачно)

### Записи дневника

- Полный текст каждой записи
- Тип ввода (голос/текст)
- Эмодзи + уровень боли
- Дата и время
- Пагинация по 20 записей

## 8. Домашние задания

### Роли

| Действие | Кто делает |
|----------|-----------|
| Создать задание | Психолог (через дашборд) |
| Выполнить / Пропустить | Клиент |
| Удалить | Психолог |

### Статусы

| Статус | Значение |
|--------|----------|
| `pending` | Ожидает выполнения |
| `completed` | Выполнено клиентом |
| `skipped` | Пропущено клиентом |

### Связь с техниками

Задание может быть привязано к конкретной технике (опционально). Если привязано — у pending-задания появляется кнопка «Начать» (открывает плеер техники). У выполненных/пропущенных — кнопки нет.

### Статистика (дашборд психолога)

| Показатель | Формула |
|------------|---------|
| Всего заданий | COUNT(*) |
| Выполнено | COUNT(status='completed') |
| Процент выполнения | completed / total * 100% |

### Бейдж на главной

На главной клиента карточка «Домашние задания» показывает бейдж с количеством pending-заданий.

## 9. Полная карта сценариев клиента

```
ГЛАВНАЯ (5 карточек)
    │
    ├── Дневник → Голос/Текст → Эмодзи → Сохранить
    │       │
    │       ├── pain < 5 → Мои записи
    │       │
    │       └── pain >= 5 → Рекомендация техники
    │               │
    │               ├── Принял → Плеер техники → Оценка после
    │               │       │
    │               │       ├── Значительно лучше (-2+) → Главная
    │               │       ├── Немного лучше / не помогло (1-я техника) → Ещё одну?
    │               │       │       ├── Да → 2-я техника (заземление)
    │               │       │       └── Нет → КБТ-флоу
    │               │       └── После 2-й техники → КБТ-флоу
    │               │
    │               └── Отказался → КБТ-флоу
    │                       │
    │                       ├── «Поговорим?» → Да → 5 вопросов → Переоценка → Фраза → Главная
    │                       └── «Поговорим?» → Нет → «Помни, я рядом» → Главная
    │
    ├── Мои записи → Список всех записей (хронология)
    │
    ├── Техники → Библиотека (дыхание, заземление, релаксация) → Плеер
    │
    ├── Домашние задания → Список (фильтры) → Выполнить / Пропустить
    │
    └── Профиль → Код-позывной, статистика (позитив), настройки режима
```

## 10. Формулы и пороги (сводка)

| Параметр | Значение | Где используется |
|----------|----------|-----------------|
| Порог предложения техники | `pain_level >= 5` | app.js (после сохранения записи) |
| Техника помогла | `pain_change <= -1` | techniques.py, profile.py |
| Значительное улучшение | `pain_change <= -2` | technique-player.js, app.js (КБТ) |
| Тренд «улучшение» | `avg_recent < avg_older - 1` | profile.py |
| Тренд «ухудшение» | Скрыт, показывается как «стабильно» | profile.py |
| Код-позывной: срок жизни | 7 дней | profile.py |
| Пагинация записей | 20 штук | dashboard.js |
| Макс. техник подряд | 2 (seq=1, seq=2) | technique-player.js |
| Допустимые pain_level | {1, 3, 5, 7, 10} | diary.py (валидация) |
